Implemented `zst_queue_element_set_pool` to optionally attach a `zst_buffer_pool_t` to a queue element. When a pool is configured, `queue_el_sink_push` now automatically acquires a new buffer from the pool, copies the data and metadata from the incoming buffer, and pushes the pooled buffer instead. This releases the original buffer back to the upstream pool immediately, avoiding upstream starvation caused by buffers held statically in the queue.
